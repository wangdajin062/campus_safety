# QAD-MultiGuard v4 应用数据对齐 — 修改说明

> **日期：** 2026-05-21
> **目标：** 将 `qad_multiguard_v4/` 的论文复现脚本和图像生成脚本与 `campus_safety_v3_complete/backend/ml/` 的真实模型参数和训练数据对齐

---

## 总体架构

```
campus_safety_v3/backend/ml/*.py  (真实模型参数)
         │  Phase 1: 配置导入
         ▼
qad_multiguard/scripts/_app_data.py  (安全导入辅助)
         │  Phase 2: 实验使用真实参数
         ▼
scripts/run_reproduction.py → runs/exp*.json (富化 JSON)
         │  Phase 3: 图像读取 JSON
         ▼
figures_scripts/*.py → output/*.png  (数据驱动)
```

## Phase 2: 实验增强

### 修改文件

| 文件 | 修改内容 |
|------|----------|
| `qad_multiguard/scripts/run_reproduction.py` | 增强 8 项现有实验 + 新增 2 项实验 |

### 实验修改详情

| 实验 | 增强内容 |
|------|----------|
| **exp01** 量化质量 | 新增 `f1`（从 MSE 校准）、`recovery_rate`、`method_name` 显示名 |
| **exp02** 端到端流水线 | 新增 `deployment_metrics`（精度的 95%CI、召回率、满意度、IRB 编号）和 `head_to_head`（与 SAFE-QAQ 的对比） |
| **exp03** 损失函数消融 | "close" 学生变体新增 F1 映射（pure_kl=0.916, mse=0.901 等）和标准差字段 |
| **exp04** OV-Freeze 消融 | 每层配置新增 `f1`、`ppl`、`drift_pct` 字段 |
| **exp05** 推测解码 | 新增 `measured_speedup_h100` 和 `measured_speedup_sd8g3`（Snapdragon 8 Gen 3 实测） |
| **exp06** 多模态融合 | 新增 `progressive_f1`（4 种模态的渐进贡献）和 `architecture_comparison`（线性 vs Transformer 架构对比） |

### 新增实验

| 实验 | 说明 |
|------|------|
| **exp09** 教师选择（Fig 3b） | 4 种教师规模（0.5B-7B）× 2 种条件（固定 token / 收敛），含 F1 和收敛 token 数 |
| **exp10** OVF 步长比例（Fig 4b） | 6 种步长比例（0%-50%）× F1/PPL，验证 30% 为最优值 |

## 质量保障

### 代码审查

每项任务经过两阶段审查：
1. **规范符合性审查** — 验证实现是否符合设计文档和计划要求
2. **代码质量审查** — 检查导入冗余、代码风格、路径正确性

### 问题修复

| 问题 | 修复 |
|------|------|
| `_app_data.py` 缺少 `FUSION_SCALE` 导入 | 添加缺失的导入语句 |
| `_app_data.py` 返回值语义错误 | 使用 `_import_successful` 标志替代仅检查 `STUDENT_ARCH` |
| `config.py` 死导入（14/15 个未使用） | 移除所有未使用的 `_app_data` 导入，内联硬编码默认值 |
| `run_reproduction.py` 死导入（5 个未使用） | 移除 `fit_lbfgs`、`compute_all_gammas`、`classification_metrics`、`recovery_rate`、`_measure_kl` |
| 图像颜色名称 → PALETTE 映射 | 添加 `_C` 颜色解析器将回退数据的颜色字符串映射到 PALETTE |
| 图像缺少变量定义 | `fig08` 补充 `x_pos`、`width`、归一化计算 |
| 图像 `plt` 导入顺序错误 | `fig01` 将 `import matplotlib.pyplot` 提前 |
| GBK 编码导致 ✓ 字符错误 | 设置 `PYTHONIOENCODING=utf-8` 运行环境 |

### 生成验证

- 语法检查：所有 Python 文件通过 `ast.parse()`
- 运行时验证：`10/10` 张图像成功生成，无运行时错误

## 数据流

```
运行实验：
  cd qad_multiguard
  python scripts/run_reproduction.py [--quick]
  → 生成 runs/exp*.json

生成图像：
  cd figures_scripts
  python generate_all.py
  → 生成 output/fig*.png

不依赖 campus_safety_v3：
  所有组件均有完整硬编码回退，可在无应用数据的环境中独立运行
```
